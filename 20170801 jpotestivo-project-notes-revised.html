JPotestivo Course Notes - Lessons 1-4

Think like a programmer
  • Procedural thinking -- perfectly clear instructions
  • Abstract thinking -- similarity and generality
  • Systems thinking -- break big problems down into smaller pieces
  • Technological empathy -- see what a computer sees
  • Debugging -- identifying causes of failure and fixing by:
    1. collecting evidence
    2. generating theories
    3. testing theories
    4. fixing problems
What is the web?
  • The web is a collection of HTML ("HyperText Markup Language") documents. Invented in the 1990s, it now counts more than 30 billion pages.
  • The major pieces are:
    1. user
    2. user's computers
    3. the Internet
    4. servers
  • HTTP is the protocol used by a browser to communicate with servers (hosts).
What is HTML?

HTML is the "heart" of the web. It consists of:

  • text content (what you see)
  • markup (what it looks like)
  • references to other documents (ex: images, videos, etc.)
  • links to other pages.

Document structure

HTML documents follow a form structure:
Document type
->Opening tag
-->Heading
--->Title (meta-data)
-->Heading
--->Body
---->Content
--->Body
->Closing tag

HTML rendering

Plain text in HTML = plain text rendered. To adjust what it looks like, use mark-up notations. Mark-up notations made of "tags" for traits like bold, italic, underline. Can add attributes to tags for anchors (i.e. "links"), to udacity, for example, and images like this: Links need to include the "http://" prefix. Images don't need closing tag. The "alt" attribute presents textual description for failures. Need to define spacing. Spacebar is a single inline space, "Break" creates a new line, and "Paragraph" creates separation via a text bundle. A break is considered an "inline" trait, whereas a paragraph is a "block." Blocks can have characteristics like height and width. "b", "br", "em", "img", and "span" are inline, whereas "p" and "div" are block.
